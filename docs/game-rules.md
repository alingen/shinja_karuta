# 信者カルタ — ゲームルール

## 対戦形式

- 1 対 1（プレイヤー対 CPU）
- 各プレイヤーは自陣に **10 枚**ずつ持つ
- 盤面合計 **20 枚**
- 先に自陣を 0 枚にした側が勝利

## 定数（変更可能）

```ts
INITIAL_HAND_SIZE = 10          // 初期手札枚数
MAX_HAND_SIZE = 15              // 手札最大枚数
MEMORIZATION_SECONDS = 20       // 暗記時間（秒）
SIMULTANEOUS_THRESHOLD_MS = 50  // 同着判定の閾値（ミリ秒）
```

## 試合の流れ

1. 暗記時間（デフォルト 20 秒）
2. ラウンド開始：音声再生
3. プレイヤーが札を選択 or 時間切れ
4. 正誤判定
5. 盤面更新
6. 勝敗判定 → 勝利 or 次ラウンドへ

## 正解処理

### 自陣の正解札を取った場合

- その札を盤面から除去
- 自陣の枚数 -1

### 敵陣の正解札を取った場合

- 敵陣の正解札を盤面から除去
- 自陣から好きな札 1 枚を選択
- 選んだ札を相手陣へ送る
- 結果として自陣の枚数 -1

## お手つき

以下はお手つきとなる。

- 正解でない札を押した
- 空札の出題時に札を押した
- 正解札が存在しない陣の札を押した

### お手つき処理

- お手つきした側が相手から 1 枚の札を受け取る（自陣 +1）
- 相手が送り札を選ぶ（CPU の場合は自動選択）

## 両者お手つき

プレイヤーと CPU の両者がお手つきした場合。

- 盤面に存在せず使用可能な札からランダムで両者に 1 枚ずつ追加
- 自陣の上限は `MAX_HAND_SIZE`（デフォルト 15 枚）を超えない
- 追加可能な札がない場合は追加なしで次ラウンドへ

## 同着

- 回答時間差が `SIMULTANEOUS_THRESHOLD_MS`（デフォルト 50ms）以内なら同着
- 正解札が置かれていた陣の所有者が取得する

| 札の場所 | 取得者   |
| -------- | -------- |
| 自陣     | プレイヤー |
| 敵陣     | CPU      |

## 空札

- 盤面に存在しない札（`isBoardEligible: false`）の音声が再生される場合がある
- 何も押さないことが正解
- 一定時間後に自動でラウンド終了
- 押した場合はお手つき

## 沼札

- 似た音声・似た場面を持つ複数の札
- `confusionGroupId` が同じ札が互いに沼札
- CPU は沼札出題時に通常より誤答率が高い

## 決まり秒

- 音声再生開始から正解札を押すまでの時間（ミリ秒記録、秒表示）
- 表示は小数第 2 位まで（例: `0.82秒`）
- 試合結果画面で「平均決まり秒」「最速決まり秒」を表示

## 試合結果

| 項目       | 内容           |
| ---------- | -------------- |
| 勝敗       | 自陣 0 枚で勝利 |
| 正解数     | 正解した枚数   |
| お手つき数 | お手つきした回数 |
| 取得札数   | 盤面から除去した枚数 |
| 平均決まり秒 | —            |
| 最速決まり秒 | —            |
